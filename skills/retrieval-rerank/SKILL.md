# 检索后重排

当检索结果过长、噪声较多、同时混有相关和不相关片段时，在回答前先做重排。如果检索已经只返回一个短而明确相关的片段，就不要额外重排。

## 适合重排的情况

以下情况适合重排：

- 检索返回了很多 chunk 或段落
- 上下文覆盖多个主题
- 用户问的是很窄的事实型问题，精度很重要
- 最终答案需要建立在最强的几个证据片段上

以下情况可以跳过重排：

- 根本没有检索结果
- 只有一个短且明显相关的片段
- 用户只要粗略总结，而当前上下文已经很聚焦

## 工作流程

1. 从已召回片段或检索上下文开始。
2. 必要时先把上下文拆成候选片段。
3. 用 reranker 根据用户问题给每个片段打分。
4. 只保留 top passages。
5. 基于重排后的证据回答。

## 回答规则

- 把 reranker 分数视为排序信号，而不是事实真伪证明。
- 用高排名片段支撑最终回答。
- 如果高排名片段仍然答不上问题，要说明证据不足。
- 不要把大量低分片段混进最终回答。
