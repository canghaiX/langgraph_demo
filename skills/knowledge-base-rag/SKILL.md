# 知识库检索问答

在调用知识库工具之前，先判断是否真的需要检索。对于闲聊、改写、翻译、基础解释和不依赖私有资料的常规推理，优先直接回答。

## 决策流程

1. 先判断请求类型。
2. 只有在答案明显依赖私有、本地、项目内或用户提供的资料时才检索。
3. 如果无需知识库也能可靠回答，就直接回答。
4. 如果不确定是否依赖知识库，优先做一次检索，不要凭空猜测。
5. 如果检索证据不足或不完整，要明确说明。

## 适合检索的情况

当用户询问以下内容时，应优先检索：

- 本地项目文档、笔记、PDF、Markdown 文件或已上传语料
- 内部事实、产品规则、制度文档或专有知识
- 明确依赖已建知识库的总结、对比、归纳类问题
- 类似“根据文档”“从知识库里看”“在这个项目里”“我们的文件怎么说”这类表述

以下情况默认不要检索：

- 问候、寒暄、一般聊天
- 翻译、转述、改写、润色
- 不依赖当前知识库的一般编程帮助
- 模型本身就能高置信度回答的常见背景知识

## 回答规则

- 把检索结果当作证据来使用，不要整段堆砌。
- 如果检索信息不完整或互相冲突，要说明不确定性。
- 优先做简洁归纳，而不是复制长段原文。
- 如果检索结果很长、很杂，应先交给重排步骤再回答。
