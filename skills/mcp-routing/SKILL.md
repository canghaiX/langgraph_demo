# MCP 路由决策

当系统同时具备 `LightRAG`、`MCP 网页搜索` 和 `MCP 本地文件系统检索` 时，先判断问题依赖哪一类信息源，再决定调用哪个工具。不要为了“保险”把所有工具都调用一遍。

## 路由规则

优先使用 `LightRAG` 的情况：

- 问题依赖已经入库的私有知识、项目资料、文档语料
- 用户明确提到“知识库”“已导入资料”“库里的文档”
- 需要基于已经建立索引的语料做问答、总结、对比

优先使用 `MCP 本地文件系统` 的情况：

- 问题依赖当前工作区里的源码、配置、README、脚本、未入库文件
- 需要列文件、读文件、搜索某个字符串在哪个文件里
- 用户明确提到“当前目录”“本地文件”“项目源码”“配置文件”

优先使用 `MCP 网页搜索` 的情况：

- 问题依赖公开网页信息、网站内容、在线文档、互联网搜索结果
- 需要获取最新网页信息或某个网站的内容
- 用户明确提到“网页”“官网”“联网查”“搜索一下”

## 冲突处理

- 如果问题更像“项目知识问答”，优先 `LightRAG`，不要先扫整个文件系统。
- 如果问题更像“找代码或看配置”，优先文件系统 MCP，不要先查知识库。
- 如果问题更像“公开互联网信息”，优先网页 MCP，不要先查本地知识。
- 如果第一选择工具没有提供足够证据，再考虑第二工具，不要并行乱查。

## 回答规则

- 先说明依据来自哪类信息源，再组织答案。
- 如果来自多个来源，要区分哪些结论来自知识库，哪些来自本地文件，哪些来自网页。
- 如果问题本身不需要工具，直接回答。
